Répertoire src

- ce répertoire doit contenir uniquement les sources .java arrangés par package ou non (pas
de fichiers d'images, ni de sons, ...)
- il est très important que src ne contienne aucun élément nécessaire à l'exécution du programme
car le répertoire src est effacé par le script d'installation des jeux  
- compilation.bat contient la commande pour compiler le programme (gestion du classpath,
des packages et de la génération des .class dans le répertoire bin)

- le répertoire devintApi contient la classe MenuAbstrait que vous devez utiliser pour créer le menu de lancement de votre jeu (dans l'avenir, on peut supposer  qu'on aura des classes abstraites pour d'autres éléments que les  menus).
- le répertoire jeu contient un exemple de jeu qui utilise les classes de menuAPI
(le fichier compilation.bat et le fichier execution.bat du répertoire bin contiennent les commandes qui permettent de compiler et d'exécuter les sources donnés en exemple dans le répertoire jeu)

Vous pouvez structurer votre programme en packages ou comme vous le souhaitez. La seule contrainte est d'utiliser la classe MenuAbstrait pour le menu de lancement du jeu et de suivre les conventions d'utilisation des touches données dans regleAPI.txt.
